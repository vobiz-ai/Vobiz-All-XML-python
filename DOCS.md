## Overview

A collection of real-world Vobiz XML call flow examples built with Python and FastAPI. Each example is a standalone server with:

- **Vobiz webhook layer** — XML responses that control the call flow

Your App

## Call Queue

> Hold music queue with round-robin agent dispatch, retry cycles and voicemail fallback

**Repos:**
- Work:     [vobiz-ai/Vobiz-Call-Queue-XML-Python](https://github.com/vobiz-ai/Vobiz-Call-Queue-XML-Python)
- Personal: [Piyush-sahoo/Vobiz-Call-Queue-XML-Python](https://github.com/Piyush-sahoo/Vobiz-Call-Queue-XML-Python)

### Use Case

Support call queue. Callers are placed on hold while the server tries available agents in round-robin order. After configurable retry cycles, falls back to voicemail. Agents register themselves via API.

### Call Flow

```
Agent App
  └── POST /agents {"number": "+91...", "name": "Raj"}  ← agent comes online

Caller → Vobiz → /answer
  └── "All agents busy. Please hold."
        └── Hold cycle 1
              ├── Play hold music + Wait 20s
              └── Dial next agent (round-robin from QueueStore)
                    ├── Agent answers → connected → /dial-complete → Hangup
                    └── No answer    → Hold cycle 2
                          ├── Play hold music + Wait 20s
                          └── Dial next agent
                                └── (after MAX_WAIT_CYCLES)
                                      └── "Please leave a message." → <Record> → Hangup

Supervisor
  └── GET /queue/status  → {callers_waiting: 3, agents_available: 2}
  └── GET /queue/metrics → {avg_wait_secs: 45, abandonment_rate_pct: 12.5}
```

## Number Capture

> Collect a caller's phone number via keypad (DTMF + # terminator) with validation and duplicate detection

**Repos:**
- Work:     [vobiz-ai/Vobiz-Number-Capture-XML-Python](https://github.com/vobiz-ai/Vobiz-Number-Capture-XML-Python)
- Personal: [Piyush-sahoo/Vobiz-Number-Capture-XML-Python](https://github.com/Piyush-sahoo/Vobiz-Number-Capture-XML-Python)

### Use Case

Lead capture or contact registration. The caller enters a 10-digit number followed by #, hears it read back, and confirms. Captured numbers are stored with duplicate detection and exportable as CSV for CRM import.

### Call Flow

```
Caller → /answer
  └── "Enter your 10-digit number followed by #. You have 15 seconds."
        └── <Gather finishOnKey="#" timeout="15">
              ├── Invalid / empty → "Not a valid number. Try again." → /answer
              └── Valid (10 digits) → Read back: "You entered: 9, 8, 7, 6..."
                    └── Gather: 1=Confirm  2=Re-enter  3=Cancel
                          ├── 1 → LeadStore.save() → "Number registered!" → Hangup
                          ├── 2 → Back to /answer
                          └── 3 → "Cancelled." → Hangup

Your App
  └── GET /leads/export.csv   → CRM import
  └── GET /leads/analytics    → {total: 234, unique: 198, duplicates: 36, today: 12}
```

## Common Architecture

Every example follows the same 3-layer pattern:

```
┌─────────────────────────────────┐
│         Your App / CRM           │  calls trigger + result APIs
│  POST /send-otp                  │
│  GET  /surveys/summary           │
└──────────────┬──────────────────┘
               │
┌──────────────▼──────────────────┐
│         server.py                │  FastAPI — both layers in one process
│  ┌─────────────────────────┐     │
│  │   Trigger / Admin API   │     │  ← your app calls these
│  └─────────────────────────┘     │
│  ┌─────────────────────────┐     │
│  │   Vobiz XML Webhooks    │     │  ← Vobiz calls these
│  │   /answer  /hangup ...  │     │
│  └─────────────────────────┘     │
└──────────────┬──────────────────┘
               │
┌──────────────▼──────────────────┐
│       *_store.py                 │  In-memory store
│   swap to Redis / Postgres       │  — same interface, just change the class
└─────────────────────────────────┘
```

## Production Notes

| Concern | Local Dev | Production |
|---------|-----------|------------|
| Public URL | ngrok (auto-start) | Set `PUBLIC_URL=https://your-server:8000` |
| Storage | In-memory dict | Replace `*_store.py` with Redis / Postgres |
| Server | `uvicorn` single process | `uvicorn --workers 4` + Redis for shared state |
| Secrets | `.env` file | Env vars injected at runtime (no `.env` file) |
| Deployment | `python server.py` | Docker + `docker-compose.yml` |
